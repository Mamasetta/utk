Итоговая верстка (html, css, js, img) собирается из папки src в которой используются технологии: lessCss, html-templating, js, сжатие изображений без потери качества, локальный сервер для тестирования.

Чтобы самостоятельно вносить изменения в исходный код шаблонов страниц, стилей и js необходимо запустить проект, для этого необзодимо:


Node v6.3.0 качать: https://nodejs.org/dist/v6.9.1/

Из корня папки проекта нужно установить зависимости npm: "npm install" (в командной строке)
Также нужно установить gulp: "npm install gulp"
Также нужно установить bower: "npm install bower"
Далее нужно установить bower-зависимости: "bower install"

Запуск и билд осуществляется с помощью команды: "gulp"
После этого запускается локальный сервер (хост и порт будут указаны в консоли), необзодимо зайти на host:posrt/page.html
увидим список страниц.

Устройство папок проекта:
src - исходники. При запущенном сервере любое изменение внутри папки src пораждает изменение итоговой папки build, куда автоматически кладутся готовый html, css, js, img - файлы. 

Точки входа:
страницы инклудят в себя template/header.html и template/footer.html
точка сборки стилей: styles.less
точка сборки js: src/js/main.js

правила сборки биллда описаны в gulpfile.js
* обратить внимание на sourcemap (это помогает ориентироваться в less файлах в процессе dev, но из-за экономии места, нужно убирать из production)
